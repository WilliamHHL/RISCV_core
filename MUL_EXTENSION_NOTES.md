# RV32M/Zmmul combinational multiplier patch

This workspace implements the multiply subset of the RISC-V M extension in the EX stage:

- `MUL`
- `MULH`
- `MULHSU`
- `MULHU`

The implementation is intentionally simple: a single-cycle combinational multiplier inside `src/EX.v`. This is useful for cycle-count/CoreMark experiments, but it will create a long EX-stage timing path in synthesis.

`DIV`, `DIVU`, `REM`, and `REMU` are **not implemented** in this package yet. For CoreMark, use one of these build targets:

```sh
./run_coremark_mul.sh rv32im-mul
```

This compiles with `-march=rv32im_zicsr -mno-div`, so GCC may emit multiply instructions but should keep division in software/libgcc.

If your GCC supports Zmmul, this is cleaner for multiplier-only hardware:

```sh
./run_coremark_mul.sh zmmul
```

Baseline comparison:

```sh
./run_coremark_mul.sh rv32i
```

After each run, check the CoreMark UART output and the log file generated by the Makefile target:

- `coremark_rv32i_baseline.log`
- `coremark_rv32im_combmul.log`
- `coremark_rv32i_zmmul_combmul.log`

Important correctness check: if you use `rv32im-mul`, disassemble `coremark/coremark.elf` and verify that no `div`, `divu`, `rem`, or `remu` instructions are present.
